考虑公司使用lua脚本的用处

Lua的特点

Lua在游戏里的应用

哪里可以使用lua

1. expression可以利用lua，不用自己写parser了

实现思路

配置文件： object = and ( obj_a, obj_b )

代码： 把obj_a, obj_b计算出来，再利用lua执行代码 " lua_set(obj_a,..), lua_and ( obj_a, obj_b )"

而要用c实现lua的lua_and, lua_set

 

Lua特点

协程，元表，弱引用，Luajit，与C/C++交互方便，高性能，开源, 函数可以返回多个值, 一流的函数对象和闭包的支持, table可以拆包成参数列表，参数列表可以直接使用{ ... }来包装成一个表, 同时赋值多个变量，local a, b, c= 1, 2, 3 或者 local x, y = y, x元表允许你实现面向对象的操作, 简单而强大的调试库 , for语句的简化，比如for i = 1, n do // do something end, 通过select函数直接从参数列表中取出某个index的参数, 尾递归优化, 增量式的垃圾回收机制,

 

缺点

Lua本身太过简单的标准库，缺失的容器库，网络库，数据库连接库

 

Lua和C++接口的部分，使用了tolua++

 

Lua在游戏中的应用：

写逻辑? UI

因为游戏这个行业需求变化快也变化多,经常每周都要更新版本出去.这样的话,如果使用编译型语言,开发效率慢很多.

 

使用方式：对于美术设计师和企划来说，他们可以字节写脚本修改ui而不需要程序员编译

 

Lua在俺们程序里的用处：

-实现某些速度要求不高的消息处理，可以自动实现序列化反序列化，用脚本实现处理内容

-实现某些ui，比如learn，calibration，setting

-实现读取配置文件 （优点：可以做语法检查，减少出错，字节码快速）
